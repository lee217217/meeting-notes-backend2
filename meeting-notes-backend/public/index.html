UI language vs output language

This index supports bilingual interface switching while keeping the result language as a separate option in the form. That means a user can browse in English and still ask the workflow to return Traditional Chinese output, or the other way around.

What this index already supports

The UI keeps language selection separate from the API payload used for generated meeting content.

Interface switcher

Header, section copy, labels, helper text, and status messages switch between English and Traditional Chinese.

Stable backend contract

The submit payload still uses the output language field for agents, so you do not need backend changes just for UI bilingual support.
